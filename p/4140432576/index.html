Java与Python的区别

发表于 2018-02-12 | 更新于 2018-02-12 | 分类于 Java ， Python | 阅读次数
字数统计 604 | 阅读时长 2

摘要

本文主要从类型、代码风格、面向对象、线程锁四个方面分析Java和Python的区别。

类型

首先简单介绍一下强类型、弱类型、静态类型和动态类型。

  • 强类型：禁止错误类型的参数继续运算，不允许隐式类型转换。
  • 弱类型：指一个语言可以隐式的转换类型（或直接转型）。

举例来说，在JS中，计算'3' / 3可以得到1。因为在计算前进行了隐式类型转换，字符串3先被转化成整数3，然后再进行除法运算。 但是在Java或Python中，若计算'3' / 3，会得到一个类型错误，提示无法在字符串中进行除法运算。因为强类型不允许隐式类型转化。

  • 静态类型：编译时就知道变量类型的是静态类型，报的错误是语法错误；
  • 动态类型：运行时才知道一个变量类型的叫做动态类型，报错时是运行错误。

其中Java属于静态类型，使用变量前必须声明数据类型；Python属于动态类型，使用前不用声明类型。

代码风格

Java中使用括号对表示类或方法的起始和结束，Python则用缩进代替括号。

面向对象

Java基本上是类/结构操作。而Python可以以独立的函数模块来处理逻辑而不需要放到类中。

线程锁

Python设计之初为了数据安全加入了GIL。GIL的全称是Global Interpreter Lock(全局解释器锁)。这里需要注意的一点是并不是所有的Python解释器都含有GIL，例如Jython就没有GIL。但是由于我们通常使用的都是CPython，因此默认Python是存在GIL的。

Python在多线程下，线程需要执行之前必须获得GIL，类似于“许可证”，并且在一个Python进程中，GIL唯一。这种机制导致的最大问题在于Python多线程程序并不能利用多个CPU的优势。例如一个使用多线程的计算密集型的任务就只能在一个单CPU上运行。但是对于I/O密集型的任务，多线程能够有效提升效率。

Java多线性的锁粒度要细很多，支持多线程并发操作，能很好的控制资源的的共享。
